# LICAN EVENTS - Project Rules

## 1. Objetivo del proyecto
LICAN EVENTS es una web profesional, escalable y modular para presentar eventos, servicios, marcas, experiencias y oportunidades de negocio.

## 2. Estructura de páginas
La web debe organizarse con las siguientes páginas principales:

- Home
- Eventos
- Servicios
- Nuestras Fiestas
  - FERAL
  - Headbang Dealers
  - Night Of Series
- Colaboradores
- Lanza tu Carrera
- Galería
- Contacto

## 3. Arquitectura de contenido
- Cada página debe ser independiente y preparada para crecer con secciones propias.
- Las rutas deben mantenerse limpias y consistentes.
- La estructura debe favorecer la reutilización de componentes comunes.
- El contenido debe organizarse por módulos: hero, destacados, servicios, eventos, testimonios, CTA, galería, contacto.

## 4. Identidad visual
La marca debe transmitir una estética:
- Oscura
- Premium
- Underground
- Púrpura
- Azul eléctrico
- Cian neón
- Alto contraste

## 5. Reglas de imágenes
- Usar imágenes reales de eventos cuando sea posible.
- Priorizar escenas con público, cabina, humo, luces y DJs.
- Preferir planos potentes, dinámicos y con impacto visual.
- Evitar imágenes borrosas, excesivamente oscuras o poco legibles.
- No usar imágenes externas salvo indicación expresa.
- Todas las imágenes deben cargarse desde public/images.

## 6. Reglas de tono
- Para Servicios y Colaboradores: tono profesional, claro, elegante y confiable.
- Para fiestas y experiencias: tono potente, directo, emocional y envolvente.
- El lenguaje debe sonar como marca cultural y premium, no genérico.

## 7. Reglas de desarrollo
- Mantener la arquitectura modular y escalable.
- Separar contenido, lógica y presentación.
- Priorizar componentes reutilizables y una estructura limpia.
- Preparar la web para crecimiento futuro sin reescribir la base.
